## Regex 正则表达式
参考文献：
    1. https://blog.csdn.net/weixin_44435110/article/details/144034153  
完成了文档获取的功能，现在就是需要将Tika从pdf中获取的String类按正则规则，将题目提取出来
- `()` 分组捕获 -> 把括号里匹配到的内容单独提取出来
  - 括号里的内容，通过matcher.group(i)去获取
    - 第 1 个左括号 ( → 对应 group(1)
    - 第 2 个左括号 ( → 对应 group(2)
    - 第 3 个左括号 ( → 对应 group(3)
    - ……
  - group(0) 特殊值
    - group(0) = 本次匹配成功的 完整内容（从头到尾符合正则规则的全部文本）；
  - 必须先执行 matcher.find()，才能调用 group(n)
  - $1,$2反向引用
    - 严格对应前面正则中的 第 1 个捕获组 (\\d+) ；
    - 含义：引用并复用 捕获组中匹配到的「所有连续数字」 ，简单说就是「原数字是什么，替换后还是什么，完全保留不变」。
    - ✅ 补充：如果正则中有多个捕获组，会有 $2、$3，依次对应第 2、3 个括号匹配的内容。
    - ```java
         .replaceAll("\\s*([A-E])\\s*[\\.、，,]\\s*", "$1")
        ```

- `.` 正则通配符 ->  匹配 任意 1 个字符（换行符除外），可以是汉字、数字、字母、标点、空格等；
- `*` 量词 → 匹配「前面的规则」0 次或无限多次，也就是 . 能匹配任意长度的字符；
- `?` 这里是「非贪婪模式」开关 → 作用是 匹配到「下一个规则」就立刻停止，不做多余匹配。
- '[]' 或 的意思
> \s 匹配 所有 Unicode 空白字符，包含：[ \t\n\r\f\v] + 全角空格 + 不间断空格 等
